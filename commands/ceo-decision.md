# Decision Framework

Apply the right decision-making framework to a CEO dilemma.

## Input
Ask: "What decision are you facing? Describe the situation and options."

## Step 1: Classify the Decision
- **Type 1 (one-way door):** Irreversible or nearly so. Examples: choosing co-founder, selling company, core architecture, major pivot.
- **Type 2 (two-way door):** Reversible, low blast radius. Examples: pricing test, marketing channel, hiring a contractor.

If Type 2: decide now with ~70% of info. Don't overthink.
If Type 1: proceed to deeper analysis.

## Step 2: Run Pre-Mortem (for Type 1 decisions)
"Assume this decision was made and it failed spectacularly 12 months from now. What went wrong?"
- List 5-10 plausible failure modes
- For each: how likely? how preventable?
- Build mitigations into the plan

## Step 3: Apply Inversion
"What would guarantee this decision fails?"
- List the behaviors/choices that would ensure failure
- Check: are any of those things currently true or likely?

## Step 4: Second-Order Analysis

## Step 5: Recommendation
Based on the analysis:
- **Decision:** [recommended choice]
- **Confidence:** [High/Medium/Low]
- **Key risk:** [the biggest thing that could go wrong]
- **Mitigation:** [how to reduce that risk]
- **Reversibility:** [how to undo this if it's wrong]
- **Review date:** [when to reassess this decision]
